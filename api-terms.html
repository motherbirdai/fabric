API Terms of Use

Last updated: February 17, 2026

These API Terms of Use ("API Terms") supplement the Fabric Terms of Service and govern your use of the Fabric application programming interfaces, SDKs, and developer tools (the "API"). In the event of a conflict between these API Terms and the Terms of Service, these API Terms shall prevail with respect to API usage.

1. API Access and Authentication

Access to the API requires a valid API key issued through the Fabric dashboard. You agree to:

  • Keep your API keys confidential and not share them with unauthorised parties
  • Include your API key in the x-api-key header or Authorization: Bearer header as specified in the documentation
  • Immediately revoke and rotate any API key you suspect has been compromised
  • Not embed API keys in client-side code, public repositories, or any publicly accessible location
  • Use separate API keys for development, staging, and production environments

API keys are issued to you and may not be transferred, sublicensed, or shared with third parties without our written consent.

Repeated intentional violation of rate limits may result in temporary or permanent suspension of API access.

3. Service Levels

Fabric aims to maintain high availability for the API but does not guarantee specific uptime percentages except as set forth in an Enterprise service agreement. Planned maintenance will be communicated at least 48 hours in advance where possible. Status and incident information is available at our status page.

The API is provided "as is" and we do not warrant that it will be error-free, uninterrupted, or meet your specific requirements. Response times may vary based on network conditions, provider availability, and request complexity.

Fabric provides official TypeScript and Python SDKs. These SDKs are provided under the terms of the MIT License as set out in the source repository. You may use the SDKs to integrate with the API in accordance with these API Terms. Fabric is not responsible for third-party or community-maintained SDK implementations.

5. MCP Integration

The Fabric API supports the Model Context Protocol (MCP) for AI agent tool discovery and execution via the /mcp/tools and /mcp/execute endpoints. When using MCP integration:

  • You are responsible for the actions taken by AI agents using your API credentials
  • Implement appropriate safeguards, spending limits, and oversight for autonomous agent operations
  • AI agents routing through Fabric are subject to all terms of this agreement including the Acceptable Use Policy
  • You acknowledge that AI agents may make routing and payment decisions autonomously and you accept liability for those decisions

6. Data Handling

When using the API:

  • Request and response data is logged for operational purposes (see our Privacy Policy)
  • You are responsible for complying with applicable data protection laws when transmitting personal data through the API
  • Do not transmit sensitive personal data (e.g., government IDs, health records) through the API unless the selected provider is specifically designed for such data
  • Fabric may cache certain API responses for performance optimisation

7. Versioning and Deprecation

The API is versioned (currently v1). When we release a new major version:

  • The previous version will be supported for at least 6 months after the new version is released
  • Deprecation notices will be communicated via email and dashboard notifications
  • Breaking changes will only be introduced in new major versions
  • Minor updates, bug fixes, and new endpoints may be added to the current version without notice

8. Restrictions

In addition to the restrictions in the Acceptable Use Policy, you agree not to:

  • Use the API to build a competing service that replicates Fabric's core functionality
  • Resell API access without a written reseller agreement
  • Use the API for benchmarking or comparative testing intended for publication without our written consent
  • Misrepresent your application's relationship with Fabric (e.g., implying official endorsement)
  • Cache trust scores or provider data beyond the cache headers specified in API responses

9. Suspension and Revocation

We may suspend or revoke your API access at any time if:

  • Your usage violates these API Terms, the Terms of Service, or the Acceptable Use Policy
  • Your usage poses a security risk to the platform or other users
  • Your account has outstanding unpaid fees
  • Required by law or regulation

Where possible, we will provide notice before suspension except in cases of imminent security risk or legal requirement.
