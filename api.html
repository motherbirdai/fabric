Use cases
What you can build
Fabric works anywhere an AI agent needs to discover, evaluate, or pay for a service.
01

Autonomous agent routing

Your agent needs image generation, web search, or code execution. Call route() and Fabric discovers providers, scores trust, picks the best match, pays, and returns the result. One function call replaces an entire integration layer.

02

Supply chain monitoring

You depend on five AI services. Fabric's webhooks alert you when a dependency's trust score drops ‚Äî new CVE, uptime degradation, abandoned repo. Know before your users do.

evaluate ¬∑ webhooks ¬∑ score-history
03

Trust badges

You're a provider. Claim your profile on the Trust Index, and embed your Fabric trust score on your docs or landing page. Verified, real-time social proof for any MCP service.

claim-profile ¬∑ trust-index ¬∑ embed
04

Cost optimisation

Multiple providers can fulfil the same request. Set a minimum trust threshold and Fabric routes to the cheapest provider above it. Cut API costs without compromising safety.

route ¬∑ cost-weight ¬∑ threshold
05

Failover & redundancy

Primary provider goes down or scores drop below your threshold? Fabric automatically re-routes to the next best alternative. No downtime, no manual intervention, no trust compromise.

route ¬∑ failover ¬∑ health-check
06

Agent-to-agent delegation

Multi-agent workflows where Agent A delegates to Agent B. Fabric verifies identity and trust at every handoff ‚Äî preventing compromised or impersonated agents from entering the chain.

evaluate ¬∑ identity ¬∑ multi-agent

MCP server allow/blocklists

Enforce organisation-wide policies on which MCP servers your team's agents can connect to. Fabric maintains the list, scores update in real time, and blocked tools never load.

Pre-install CI/CD gate

Marketplace trust layer
